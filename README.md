# Арматурина

Мега смешной Telegram-бот для групп. Работает только с админами: если админ пишет фразу со словом `Арматурина` и `@username`, бот отвечает рандомным стикером или GIF на последнее сообщение указанного пользователя.

## Запуск

1. Создай бота через BotFather и выключи ему privacy mode: `Bot Settings` -> `Group Privacy` -> `Turn off`.
2. Скопируй `.env.example` в `.env` и укажи `ARMATURINA_BOT_TOKEN`.
3. Установи зависимости и запусти:

```bash
npm install
npm start
```

Если в одном контейнере запускается несколько ботов, можно импортировать Арматурину в главный ESM-файл:

```js
import { armaturina } from './armaturina/src/index.js';

await armaturina();
```

`armaturina()` не блокирует запуск остальных ботов: она поднимает polling в фоне и возвращает инстанс после загрузки файла с пулом.

Если не хочешь использовать env-переменные, передай настройки явно:

```js
import { armaturina } from './armaturina/src/index.js';

await armaturina({
  token: process.env.ARMATURINA_BOT_TOKEN,
  dataFilePath: './armaturina/data/media-pool.json'
});
```

## Команды и фразы

Все команды ниже работают только в группе и только от администратора.

- `Арматурина, дай по хрептине @username`
- `Арматурина, @username`
- Ответ на сообщение: `Арматурина`
- Ответ на сообщение: `Арматурина фас`
- `/arm_help` - помощь.
- `/pool` - размер текущего пула.
- `/stats` - общая статистика ударов, доступна всем.
- `/top` - недельный топ `Почетная хрептина недели`, доступен всем.
- `/mute N` ответом на сообщение - мут на `N` минут.
- `/mute @username N` - мут пользователя на `N` минут, если бот уже знает его user id.
- `/ban` ответом на сообщение - бан пользователя с удалением его сообщений.
- `/ban @username` - бан пользователя с удалением его сообщений, если бот уже знает его user id.
- `Арматурина завари ебало на N минут` ответом на сообщение - мут.
- `Арматурина завари ебало N` ответом на сообщение - мут.
- `Арматурина уеби его` ответом на сообщение - бан.
- `/addstickerpack <pack_name>` - добавить стикерпак по имени.
- `/addultrastickerpack <pack_name>` - добавить стикерпак в ultra-пул.
- `/retag [regular|ultra|all] [limit]` - протегировать уже добавленные медиа через Gemini.
- `Арматурина, добавь стикерпак <pack_name>` - то же самое обычной фразой.
- `Арматурина, добавь ультра стикерпак <pack_name>` - добавить стикерпак в ultra-пул обычной фразой.
- Ответить на стикер: `Арматурина, добавь стикерпак` - добавит пак этого стикера.
- Ответить на GIF/animation: `Арматурина, добавь гифку` или `/addgif` - добавит GIF.
- Ответить на GIF/animation: `Арматурина, добавь ультра гифку` или `/addultragif` - добавит GIF в ultra-пул.

## Статистика и ultra-удар

Каждый успешный удар записывается в `data/media-pool.json`. `/stats` показывает общую статистику чата, а `/top` показывает недельный топ по количеству полученных ударов.

`Почетная хрептина недели` - это лидер текущей ISO-недели. Новая неделя начнет считаться отдельно, старая статистика останется в JSON.

Ultra-удар имеет шанс 1% при обычном ударе. Его нельзя вызвать командой вручную: бот сам решает, когда он прокнул. Если ultra-пул пустой, бот всегда использует обычный пул.

Когда ultra-удар выпадает, бот сначала отправляет редактируемое сообщение с зарядкой мега-хрептины: зеленые квадратики и проценты до 100%. На 100% появляется `Лови маслину`, сообщение удаляется, затем отправляется ultra GIF/стикер.

## Gemini-теги

Gemini анализирует стикеры/GIF при добавлении и сохраняет теги в `data/media-pool.json`. При ударе бот анализирует контекст сообщения и пытается выбрать медиа с подходящими тегами. Если Gemini недоступен или совпадений нет, используется обычный случайный выбор.

Настройки:

```bash
ARMATURINA_GEMINI_API_KEY=
ARMATURINA_GEMINI_MODEL=gemini-3.5-flash
ARMATURINA_GEMINI_MAX_TAGS_PER_PACK=30
ARMATURINA_GEMINI_DEBUG=false
```

Для диагностики можно включить:

```bash
ARMATURINA_GEMINI_DEBUG=true
```

Тогда в консоли будут сырой ответ Gemini, `finishReason`, safety ratings и итоговые теги.

Для static `.webp` стикеров бот анализирует сам файл. Для animated `.tgs`, video stickers и GIF сначала используется Telegram thumbnail. Если thumbnail нет, медиа останется без точных визуальных тегов до более тяжелой конвертации.

Старый пул можно догнать командой:

```text
/retag all 30
/retag regular 20
/retag ultra 10
```

## Хранение

Пул стикеров/GIF хранится в JSON-файле `data/media-pool.json`. После добавления стикерпака или GIF бот сразу сохраняет файл, поэтому перезапуск процесса не очищает пул.

Путь можно поменять в `.env`:

```bash
DATA_FILE=data/media-pool.json
```

Последние сообщения пользователей хранятся только в памяти процесса. После рестарта бот снова начнет запоминать последние сообщения по мере активности в группе.
